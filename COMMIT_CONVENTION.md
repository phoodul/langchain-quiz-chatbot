# Commit Convention

이 프로젝트는 협업과 히스토리 관리를 위해 **Conventional Commits** 스타일을 따릅니다.

## 1. 메시지 구조
커밋 메시지는 다음과 같은 기본 구조를 가집니다.

```
<type>: <description>

[body] (선택 사항)
[footer] (선택 사항)
```

## 2. 커밋 타입 (Type)
| 타입 | 설명 |
| :--- | :--- |
| **기능** | 새로운 기능 추가, 디자인 변경 |
| **수정** | 버그 수정 |
| **문서** | 문서 수정 (README.md, Docstring 등) |
| **개선** | 코드 리팩토링, 포맷팅, 스타일 변경 |
| **설정** | 패키지 관리 (`uv`), 빌드 설정, 테스트 추가 등 |

## 3. 작성 규칙
- **언어**: 모든 내용을 **한국어**로 작성합니다.
- **제목**: 50자 이내로 요약하며 마침표(`.`)를 찍지 않습니다.
- **어조**: "합니다", "함" 등의 명령문 또는 명사형 종결 어미를 사용합니다.
- **구분**: 제목과 본문은 한 줄을 띄워 구분합니다.

## 4. 예시 (Examples)
- `기능: LangGraph 기반 퀴즈 생성 워크플로우 및 UI 구현`
- `수정: API 호출 타임아웃 오류 수정`
- `문서: README.md 및 agents.md 업데이트`
- `개선: 유틸리티 함수 구조 변경 및 코드 포매팅 적용`
- `설정: uv add langchain-openai 패키지 설치`
