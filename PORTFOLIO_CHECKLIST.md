# 📝 LangChain Quiz Chatbot 포트폴리오 체크리스트

이 문서는 `docs/archive/DONE_포트폴리오_가이드.md`의 지침을 바탕으로, 본 프로젝트를 SW특기자 전형 등 포트폴리오로 활용하기 위해 개발 과정에서 반드시 기록하고 관리해야 할 항목들을 정리한 것입니다.

## 1. 문제 정의 및 목표 (Problem Definition)
- [ ] **개발 동기**: 왜 '퀴즈 챗봇'을 만들기로 했는가? (예: 학습 효율 증진, 학습 데이터 자동화 등)
- [ ] **페르소나/타겟**: 이 서비스는 누구를 위한 것인가?
- [ ] **차별점**: 기존 LLM 서비스나 일반 챗봇과 비교했을 때 이 프로젝트만의 특징이나 강점은 무엇인가?

## 2. 설계 및 기술 스택 (Architecture & Tech Stack)
- [ ] **사용 기술**: Python 3.12+, LangChain (LCEL), LangGraph, `uv` 등을 선택한 기술적 이유
- [ ] **시스템 아키텍처**: 전체 데이터 흐름도 작성 (Input -> Prompt -> LLM -> Output Parser -> Quiz Output)
- [ ] **LangGraph 활용**: 챗봇의 상태(State)를 어떻게 설계했고, 노드 간의 전이를 어떻게 정의했는지 기록

## 3. 핵심 구현 및 코드 설명 (Core Implementation)
- [ ] **프롬프트 엔지니어링**: 고품질 퀴즈 생성을 위해 적용한 프롬프트 기법 (Few-shot, Chain-of-Thought 등)
- [ ] **구조화된 출력**: LLM의 응답을 JSON 등 정해진 형식으로 받기 위해 수행한 `Output Parser` 설정
- [ ] **타입 안정성 & 비동기**: 모든 함수에 적용된 `Type Hinting`과 I/O 성능 향상을 위한 `async/await` 구조 설명

## 4. 과정 중심의 기록 (Process & Improvement)
- [ ] **트러블슈팅**: 개발 중 발생한 문제(예: 환각 현상, 파싱 에러)를 어떻게 해결했는지 구체적인 사례 기록
- [ ] **성능 개선 시도**: 초기 버전보다 퀴즈의 정확도나 속도를 높이기 위해 시도한 방법들
- [ ] **버전 관리**: 주요 기능 추가나 리팩토링 과정에서의 고민 흔적 남기기

## 5. 한계점 및 개선 방향 (Self-Reflection)
- [ ] **현재의 완성도**: 현재 시공된 기능의 명확한 한계점 인식
- [ ] **향후 고도화**: RAG(외부 문서 참조) 적용, 멀티 모달 퀴즈(이미지 포함) 등 확장 가능성 제시

## 💡 포트폴리오 작성 핵심 원칙
> **"코드보다 설명, 결과보다 과정, 성공보다 개선 시도"**
- 단순히 "돌아가는 코드"를 만드는 데 그치지 않고, **왜** 이렇게 짰는지에 대한 논리를 증명하는 것이 핵심입니다.
- 개발 중 막히는 부분이 있다면 그 과정 자체를 캡처하거나 문서로 남겨두세요. 그것이 가장 강력한 포트폴리오 재료가 됩니다.
